# 迁移文档导出功能
■ 查看全部对话的ui改成“迁移文档导出”button样式与底盘配置的一样

■ 点击button后依旧出现弹窗，参考该文件夹下图片生成内容ui

■ 然后需要新开一个session，来总结主对话所有的内容生成项目经验

■ 然后用户前端选择生成哪些内容

■ 之后发送给session让其总结生成md内容，可以在前端预览

■ 这个session先做成“前端获得生成完的结果就删除”的那种

■ 预览完后用户可以选择导出，或者复制内容，或者进行编辑



| **内容类型** | **AI 检索什么** | **总结成什么** |
| --- | --- | --- |
| **任务目标** | **用户最初请求、任务标题、Agent 对任务的理解** | **本轮要解决的问题、交付目标、任务边界** |
| **当前进度** | **测试结果、完成状态、执行结束总结、未完成事项** | **本轮完成了什么、还剩什么、当前项目状态** |
| **稳定规则** | **用户明确要求、项目约束、不能修改的内容、验收标准** | **后续任务必须遵守的规则和限制** |
| **关键决策** | **方案选择、Agent 判断依据、用户确认或否定的方案** | **为什么采用某个方案，放弃了什么方案** |
| **可复用经验** | **成功排查路径、修复模式、调试方法、可迁移操作** | **下次类似问题可以复用的经验** |
| **验证证据** | **测试命令、测试结果、Diff、日志、文件路径** | **支撑结论的证据链，说明为什么可信** |
| **失败路径** | **报错、失败测试、无效尝试、被推翻的假设** | **哪些路走不通，避免后续重复尝试** |
| **风险待确认** | **未解决问题、模糊结论、高风险修改、需要人工判断处** | **下一轮或研究者需要确认的事项** |
| **下一轮提示** | **当前状态、下一步建议、需要带入新会话的信息** | **可直接复制到下一轮对话的启动上下文** |
| **技能库条目** | **可抽象为长期方法论的经验、跨项目可复用知识** | **可沉淀到知识技能库的结构化条目** |
